# แผนงาน V7: ตรวจเลขใบขนซ้ำ (dedup) + แจ้งเตือน

สถานะ: ✅ ทำเสร็จ + unit-test logic ผ่าน (ยังไม่ deploy)

## ขอบเขตงาน (จากผู้ใช้)
เมนูรายงาน ตอนอัปโหลดรูป ถ้าเลขใบขนสินค้าตรงกัน (ซ้ำ) ให้:
1. แจ้งเตือน
2. แสดงข้อความแค่ใบขนเดียว (ไม่แสดงซ้ำ)

## บริบท
- Logic อยู่ฝั่ง **API (server)** — `api/gemini.js` block `type === 'team'`
- Gemini คืน `declarations[]` แต่ละใบมี `no` (เลขใบขน A000-00000-00000)

## การตัดสินใจ (checkpoint)
- **Q1 → (B) keep-first** — เจอ `no` ซ้ำ เก็บใบแรก ทิ้งใบที่เหลือ
  (ไม่เลือก merge; นิยาม "ซ้ำ" = `no` ตรงกันหลัง trim ช่องว่าง, ข้ามใบที่ `no` ว่าง)
- **Q2 → (A)** แจ้งเตือนซ้ำ **รวมในกล่องเหลืองเดิม** (warnings[]) — ไม่เพิ่ม UI ใหม่
- Auto (ผมตัดสินใจเอง): จำนวน "N ใบขน" + เลขลำดับ 1,2,3 + เช็คข้อมูลขาด → คำนวณจากใบที่เหลือหลัง dedup

## สิ่งที่แก้ (api/gemini.js, block team)
- เปลี่ยน `const decls = parsed.declarations` → dedup ก่อน:
  - วน `rawDecls`, normalize key = `(d.no||'').trim()`
  - ถ้า key ไม่ว่าง + เคยเห็นแล้ว → เก็บลง `dupNos`, ข้าม (ไม่ push)
  - ถ้า key ว่าง → เก็บไว้ (dedup ไม่ได้)
- `warnings` ขึ้นก่อนด้วยบรรทัดซ้ำ: `เลขใบขน {no} ซ้ำ — แสดงครั้งเดียว` (ต่อ 1 เลขซ้ำ)
- จากนั้นเช็คข้อมูลขาด (REQUIRED เดิม) บน `decls` ที่ dedup แล้ว
- `count` + text ใช้ `decls` ที่ dedup แล้ว

## ผล unit-test (Node, จำลอง logic)
Input: 4 ใบ — #1,#3 เลข 05461 เหมือนกัน; #2,#4 เลข 06732 (#4 มีช่องว่างนำ/ตาม)
- count = 2 ✓
- warnings: "เลขใบขน A024-06906-05461 ซ้ำ...", "เลขใบขน A024-06906-06732 ซ้ำ..." (trim จับช่องว่างได้) ✓
- keep-first: ใบ 05461 เก็บใบแรก (qty ว่าง) → text แสดง qty ว่าง + เตือน "ขาด: จำนวน" ✓
- เลขลำดับ 1,2 + "จำนวน 2 ใบขน" ✓
