# แผนงาน V5: ปรับหน้ารายงานตรวจทีม (header / มุมโค้ง / auto-grow / คำเตือน)

สถานะ: ✅ ทำเสร็จ + verify ในเบราว์เซอร์แล้ว

## ขอบเขตงาน (จากผู้ใช้)
1. เมนูรายงาน: เอาคำว่า "ข้อความรายงาน" + เครื่องหมายถูก (✓) ออก
2. กล่องข้อความรายงานให้โค้งเท่ากันทั้ง 4 มุม (เดิมมุมซ้ายล่างไม่โค้ง)
3. ข้อความรายงานเห็นไม่ครบ — ด้านล่างถูกตัด (เห็นแค่บางส่วน)

## การตัดสินใจ (checkpoint)
- **Q1 → (A)** ลบ header ทั้งแถบ (✓ + "ข้อความรายงาน") — กล่องคำเตือนกลายเป็นส่วนบนสุดของการ์ด
- เพิ่มเติมจากผู้ใช้: ให้ **เริ่มต้นเป็นคำเตือน** "พบข้อมูลไม่ครบบางใบขน" ครอบคลุม 5 ฟิลด์:
  เลขที่ใบขนสินค้า, สินค้า, เมืองกำเนิด, จำนวน, น้ำหนัก
- **Q2 → (A)** เพิ่ม `no` (เลขที่ใบขนสินค้า) เข้าการเช็คข้อมูลขาด (เดิมเช็คแค่ 4 ฟิลด์)
  → คำเตือน **แสดงเฉพาะเมื่อมีข้อมูลขาดจริง** (ตามเดิม)

## สาเหตุ bug ข้อ 3 (สำคัญ)
`autoGrowTeam()` เดิมถูกเรียก **ก่อน** `teamResultCard` แสดง (`display:block`)
→ ตอนวัด `scrollHeight` การ์ดยัง `display:none` → ได้ 0 → textarea ค้างที่ `min-height:120px`
→ เห็นแค่ ~3 บรรทัด ด้านล่างถูกตัด

## สิ่งที่แก้
**index.html**
1. ลบ `<div class="step-header">…</div>` (✓ + ข้อความรายงาน) ออกจาก `#teamResultCard`
   → กล่อง `#teamWarningBox` เป็น element แรกของการ์ดโดยธรรมชาติ
2. textarea `#teamChatBubble`: `border-radius:18px 18px 18px 4px` → **`border-radius:18px`** (โค้งเท่ากัน 4 มุม)
3. ย้าย `autoGrowTeam()` จากก่อน display:block → **หลัง** `teamResultCard.style.display='block'`
   - หมายเหตุ: ใช้เรียกแบบ synchronous (ไม่ใช่ requestAnimationFrame)
     เพราะการอ่าน `scrollHeight` บังคับ reflow อยู่แล้ว → วัดถูกต้องทันที
     (rAF ทดสอบแล้วไม่น่าเชื่อถือในบาง context)

**api/gemini.js**
4. `REQUIRED` เพิ่ม `no:'เลขที่ใบขนสินค้า'` เป็นตัวแรก → เช็คครบ 5 ฟิลด์
