# แผนงาน V3: ปรับ layout ประเภทสินค้า + alert + ปุ่ม Clear

สถานะ: กำลังสัมภาษณ์ (grill-me) — รอบที่ 3

## ขอบเขตงาน 3 ข้อ (จากผู้ใช้)
1. ประเภทสินค้า (หอม/กระเทียม/อื่นๆ) จัดให้อยู่บรรทัดเดียวกัน + ขยับ "ประเทศกำเนิด" ไปขวา
   ให้ตรง (aligned) กับบรรทัดล่าง (น้ำหนักหน้าใบขน | ชุดที่)
2. เอาไอคอน ✅ หน้า alert "ตรวจสอบความถูกต้อง..." ออก + แก้คำเป็น "ตรวจสอบความถูกต้อง และกรอกช่อง * ให้ครบ"
3. ปรับปุ่ม Clear ให้ขนาด+รูปแบบเหมือนปุ่ม "สร้างรายงาน"

  - **ข้อจำกัด:** บนมือถือ 3 chip (กว้างสุด "กระเทียม") ใส่ในครึ่งจอ (~170px) บรรทัดเดียว = แน่นมาก → ต้องย่อ chip

  - `#btnGenerate` "สร้างรายงาน" (บรรทัด 462): class `btn-gen` (orange gradient, radius 0.875rem=14px), padding 12px 24px, ไอคอนรูปภาพ

  - padding เท่ากันอยู่แล้ว (12px 24px); ต่างที่ radius (10 vs 14) + สี (แดง vs ส้ม gradient)

## คำถาม/การตัดสินใจ (checkpoint)

### ✅ Q1 (ข้อ 1): layout ประเภทสินค้า/ประเทศกำเนิด
**ตัดสินใจ:** คงขนาด chip เดิม (ไม่ย่อ) แต่ทำ **คอลัมน์ซ้ายกว้างขึ้น** ให้ 3 chip อยู่บรรทัดเดียว
- ปรับ **ทั้ง row 4 และ row 5** ให้ใช้ grid ratio เดียวกัน (เช่น ~1.7–1.9fr : 1fr — จูนใน browser)
  → เส้นแบ่งคอลัมน์ของสองแถวเลื่อนตามกันเสมอ (aligned)
- ซ้าย(กว้าง) = goodsChips / น้ำหนักหน้าใบขน ; ขวา(แคบ) = ประเทศกำเนิด / ชุดที่
- ทดสอบบน mobile 375px ให้ chip อยู่บรรทัดเดียวจริง แล้วค่อยล็อกอัตราส่วน

### ✅ Q2 (ข้อ 2): alert
**ตัดสินใจ:** เอาไอคอน ✅ ออก + เปลี่ยนข้อความ `#editAlert` เป็น **"ตรวจสอบความถูกต้อง และกรอกช่อง * ให้ครบ"**

### ✅ Q3 (ข้อ 3): ปุ่ม Clear
**ตัดสินใจ:** คง **สีแดง** ไว้ แต่ทำ **ขนาด/ทรง/radius เหมือนปุ่ม สร้างรายงาน** (btn-gen)
- สร้าง class `.btn-danger` = red gradient pill (เช่น 135deg #EF4444→#DC2626) + box-shadow แดง + radius 0.875rem + padding/font เท่า btn-gen — คงไอคอนถังขยะ
- ใช้กับปุ่ม Clear ทุกจุดที่ตอนนี้ใช้ inline red radius:10px เพื่อความ consistent:
  `#btnClearBefore` (ข้าง สร้างรายงาน), `#btnClearWeight` (หลังรายงาน), `#btnClearTeam` (หน้า team)
- ลบ inline `background/border-radius` เดิมออก แทนด้วย class

### ✅ Q4 (ข้อ 4 เพิ่ม): ปุ่มบันทึกรูป + สร้างใหม่
**ตัดสินใจ:** จัด 2 ปุ่มใน `.save-area` ให้อยู่บรรทัดเดียวกัน — **ซ้าย = สร้างใหม่, ขวา = บันทึกรูป** (สลับลำดับจากเดิม)
- ทั้งคู่ทรง/ขนาด/radius เหมือน btn-gen (radius 0.875rem, padding, font)
- **คงสีเดิม:** สร้างใหม่ = outline (btn-secondary), บันทึกรูป = เขียว (btn-save)
- ให้ 2 ปุ่มกว้างเท่ากัน (flex:1) อยู่บรรทัดเดียวบนมือถือ

## สรุปแผน implement (4 ข้อ) — พร้อมลงมือ

1. **Layout ประเภทสินค้า:** row 4 + row 5 ใช้ grid ratio เดียวกัน (ซ้ายกว้าง ~1.7–1.9fr : 1fr, จูนใน browser) → chips หอม/กระเทียม/อื่นๆ บรรทัดเดียว, origin/ชุดที่ คอลัมน์ขวาแคบ, เส้นแบ่ง 2 แถวตรงกัน
2. **Alert:** เอา ✅ ออก, ข้อความ = "ตรวจสอบความถูกต้อง และกรอกช่อง * ให้ครบ"
3. **ปุ่ม Clear:** เพิ่ม `.btn-danger` (red gradient pill ทรงเดียวกับ btn-gen) ใช้กับ Clear ทุกปุ่ม
4. **ปุ่มบันทึกรูป/สร้างใหม่:** บรรทัดเดียว ซ้าย=สร้างใหม่(outline) ขวา=บันทึกรูป(เขียว) ทรงเท่า btn-gen, กว้างเท่ากัน

**ทดสอบ:** mobile 375px — chips บรรทัดเดียว + 2 แถวคอลัมน์ตรงกัน, alert ข้อความใหม่ไม่มีไอคอน, ปุ่ม Clear ทรงเดียวกับ สร้างรายงาน แต่แดง, ปุ่มสร้างใหม่/บันทึกรูปบรรทัดเดียวทรงเดียวกัน
