# แผนงาน: ปรับปรุงเว็บ weight-report

สถานะ: กำลังสัมภาษณ์ความต้องการ (grill-me)

## ขอบเขตงาน 4 ข้อ (จากคำสั่งผู้ใช้)
1. ปรับรูปแบบเว็บไซต์ตามไฟล์ที่แนบ (ภาพ Financial Dashboard mockup)
2. เอาเลข "1" (หน้าอัปโหลดภาพใบชั่งน้ำหนัก) และเลข "2" (หน้าตรวจสอบและแก้ไขข้อมูล) ออก
3. เอาช่องว่างใต้ปุ่ม "สร้างรายงาน" ออก
4. เมนู "ชั่งน้ำหนัก" — ฝั่งปุ่ม "อ่านข้อมูลด้วย Gemini" เพิ่มปุ่ม "Manual" ที่ข้ามการอ่านภาพ ไปหน้า "ตรวจสอบและแก้ไขข้อมูล" เลย

## บริบทโค้ดที่สำรวจแล้ว
- โปรเจกต์: `weight-report/index.html` — single-file SPA ภาษาไทย, ใช้ font Sarabun, โทนสีส้ม (`--orange:#E8691A`), การ์ดขาวโค้งมน มือถือเป็นหลัก
- มี 2 หน้า (nav): `page-weight` (ชั่งน้ำหนัก, 3 step: อัปโหลด → ตรวจสอบ/แก้ไข → ผลลัพธ์) และ `page-team` (รายงานทีม)
- Step 1 การ์ด `#card-upload` มี `#num-1`; Step 2 การ์ด `#card-edit` มี `#num-2`
- ปุ่ม "สร้างรายงาน" คือ `#btnGenerate` อยู่ใน container ที่มี `padding-bottom:60px` (บรรทัด 421) — เป็นตัวเก็งช่องว่างที่ต้องเอาออก แต่ยังไม่ยืนยัน
- ภาพที่แนบ = เทมเพลต "Financial Dashboard" (Uizard-style): โทนดำ/ครีม/ส้มอ่อน, sidebar, การ์ด widget หลายแบบ, กราฟวงกลม, ข้อความทักทายใหญ่ — เป็นคนละ genre กับแอปชั่งน้ำหนักบนมือถือปัจจุบันโดยสิ้นเชิง

## คำถาม/การตัดสินใจ (checkpoint)

### ✅ Q1: ขอบเขตการปรับดีไซน์ตามภาพแนบ
**ตัดสินใจ:** ผสมผสาน — คงโครงหน้าแบบ step-based (1→2→3) เดิมไว้ทั้งหมด แต่รีเฟรชสไตล์ของการ์ด/ปุ่ม/ไอคอนให้ดูทันสมัยขึ้นแบบภาพที่แนบ (ไม่ทำ sidebar/dashboard เต็มรูปแบบ ไม่เปลี่ยนโครงสร้างหน้า)

### ✅ Q2: โทนสี
**ตัดสินใจ:** คงส้ม (#E8691A) เป็นสีหลัก/แบรนด์เหมือนเดิม — ปรับแค่ shadow/ความโค้งมน/spacing ให้ดูทันสมัยขึ้นตามภาพ ไม่เปลี่ยนพื้นหลังเป็นดำ/ครีม

### ✅ Q3: องค์ประกอบที่จะรีเฟรช
**ตัดสินใจ:** (1) เปลี่ยน step-num (เลข 1/2) เป็นไอคอน badge วงกลมสีอ่อนแทน — **จุดนี้ตอบโจทย์ข้อ 2 ของ user ไปในตัว (เอาเลขออก)** และ (2) เพิ่มช่องว่าง/เงาการ์ดให้นุ่มและกว้างขึ้นแบบภาพตัวอย่าง — ไม่ทำ pill button, ไม่ปรับ typography hierarchy

### ✅ Q4: ไอคอน badge ต่อ step
**ตัดสินใจ:** step1 (`#num-1`) = ไอคอนกล้องถ่ายรูป, step2 (`#num-2`) = ไอคอนเช็ค✓/แก้ไข — สีอ่อนบนพื้นหลังส้มอ่อน (โทนเดียวกับ `--orange-light`)

### ✅ Q5 (ข้อ 3 เดิม): ช่องว่างใต้ปุ่มสร้างรายงาน
**ตัดสินใจ:** ยืนยันแล้วว่าคือ `padding-bottom:60px` ที่บรรทัด 421 (แถว flex ครอบปุ่ม Clear + สร้างรายงาน ใน `#card-edit`) — ให้เอาออก/ลดให้เหลือ spacing ปกติเท่ากับ card อื่น

### ✅ Q6 (ข้อ 4): เงื่อนไขปุ่ม Manual
**ตัดสินใจ:** ปุ่ม Manual กดได้เสมอ ไม่ต้องรออัปโหลดภาพก่อน (ต่างจาก `readBtn` ที่ disabled จนกว่าจะมีไฟล์) — กดแล้วข้ามการเรียก Gemini ไปเปิด `#card-edit` ทันทีโดยไม่มีข้อมูลกรอกล่วงหน้า (ฟอร์มว่าง ให้ผู้ใช้กรอกเอง) — ถ้ามีภาพอัปโหลดไว้แล้ว ให้เก็บภาพนั้นไว้เหมือนเดิม ไม่ล้างทิ้ง

### ✅ Q7: รูปแบบปุ่ม Manual
**ตัดสินใจ:** ใช้สไตล์ btn-secondary (ขอบ พื้นขาว) วางถัดจากปุ่ม "อ่านข้อมูลด้วย Gemini" ในแถวเดียวกัน (ต่อจาก/แทนที่ตำแหน่งปุ่ม "ล้างรูป" เดิม หรือเพิ่มเป็นปุ่มที่ 3 ในแถว — ให้ตัดสินใจตอน implement ให้ layout ไม่แน่นเกินไปบนมือถือ)

## สรุปสถานะ: ครบทั้ง 4 ข้อแล้ว พร้อม implement

1. **ปรับดีไซน์ตามภาพแนบ** → คงโครง step-based, คงสีส้มเป็นหลัก, เพิ่ม: (ก) ไอคอน badge วงกลม (กล้อง/เช็ค) แทนเลข step 1/2, (ข) เงา/spacing การ์ดให้นุ่มและกว้างขึ้น
2. **เอาเลข step ออก** → ตอบโจทย์แล้วจาก Q3/Q4 (แทนที่ด้วยไอคอน ไม่ใช่ตัวเลขเปล่า)
3. **เอาช่องว่างใต้ปุ่มสร้างรายงานออก** → ลบ `padding-bottom:60px` ที่บรรทัด 421
4. **ปุ่ม Manual** → เพิ่มถัดจาก `#readBtn`, สไตล์ btn-secondary, กดได้เสมอ (ไม่ disabled), คลิกแล้ว: แสดง `#card-edit` ทันที (ฟอร์มว่าง ไม่เรียก Gemini), ไม่ยุ่งกับภาพที่อัปโหลดไว้ (ถ้ามี)

รอ user ยืนยันก่อนเริ่ม implement
