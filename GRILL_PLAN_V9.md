# แผนงาน V9: หน้ารอ (launch screen) สีส้ม + โลโก้ ตอนเปิดผ่าน web app บน iOS

สถานะ: ✅ แก้เสร็จ (client-only, ยังไม่ deploy)

## ขอบเขตงาน (จากผู้ใช้)
ตอนเข้าผ่าน web app บน iOS บางครั้งต้องรอ → อยากให้มีหน้าจอรอ
พื้นหลังสีส้ม ตรงกลางเป็นโลโก้แอพ โหลดเสร็จแล้วเด้งเข้าแอพปกติ

## ข้อเท็จจริงจากโค้ด (สำรวจแล้ว ไม่ต้องถาม)
- แอพเป็นไฟล์เดียว `index.html` (~1,616 บรรทัด) รูปฝัง base64 ทั้งหมด
  ไม่มี API call ตอนเปิด → **ช่วงรอ = รอโหลด index.html จากเน็ต** (ไม่มี service worker)
- `manifest.json`: `display: standalone`, `background_color: #F2EDE6` (ครีม),
  `theme_color: #E8691A` (ส้ม), มีไอคอน 2 ตัว (จริงๆ ไฟล์เดียวกัน 180×180 px
  แต่ประกาศเป็น 192 กับ 512 — ขนาดไม่ตรงจริง)
- ไอคอน + apple-touch-icon เป็น **data: URI base64** ทั้งคู่ (ไม่มีไฟล์รูปจริงใน repo)
- **โลโก้ = ลายเส้นหอม+กระเทียมสีส้ม บนพื้นโปร่งใส (มี alpha)**
- สีส้มหลักของแอพ: `--orange: #E8691A`

## ข้อจำกัดทางเทคนิคที่คุมการออกแบบ
- splash ที่เขียนใน index.html ขึ้นได้หลังไฟล์โหลดถึงเครื่องแล้วเท่านั้น
  → ปิดช่วงรอเน็ต (จอขาว) ไม่ได้
- iOS สร้าง launch screen ให้เองจาก `background_color` + ไอคอนใน manifest
  และโชว์ค้างจนหน้าเว็บ paint แรก → ตรง requirement พอดี
- iOS จำ manifest ตอน Add to Home Screen → ต้องลบแล้วเพิ่มไอคอนใหม่จึงเห็นผล
- ไอคอนโปร่งใส: iOS เติมพื้น **ดำ** ให้เอง + โลโก้ส้มบนพื้น splash ส้มจะกลืนกัน

## การตัดสินใจ (checkpoint)
- **Q1 → หน้ารอสีส้มอย่างเดียว** — ใช้ native launch screen ของ iOS
  (เปลี่ยน `background_color` เป็นส้ม) ไม่ทำ service worker / ไม่ทำ overlay JS
  เหตุผล: ครอบคลุมช่วงรอทั้งหมด แก้บรรทัดเดียว ความเสี่ยงต่ำสุด
- **Q2 → ไอคอนโฮมปัจจุบัน = โลโก้บนพื้นขาว/ครีม** — ตรวจไฟล์แล้ว: PNG พื้นขาว
  opaque ทุกพิกเซล (ไม่ใช่โปร่งใส) → data: URI ใช้ได้จริงบนเครื่องผู้ใช้
  และ **ไม่ต้องทำไอคอนใหม่** — splash จะเป็นไอคอนขาว (มีโลโก้) ลอยบนพื้นส้ม เห็นชัด
- Q4 (ทำไอคอน splash แยก) → ตกไป ไม่จำเป็นแล้วจากผล Q2

- **Q3 → ใช้ #E8691A** (ส้มหลักของแอพ = theme_color เดิม) โทนเดียวกับแถบ topbar
- **Q5 → มีหลายคนใช้** — ต้องแจ้งทีมทุกคน re-add ไอคอน (ดูขั้นตอนด้านล่าง)
- **Q6 → แก้เฉพาะสี splash** — ไม่แตะ metadata ขนาดไอคอน (ประกาศ 192/512 แต่จริง 180)
  เพราะของเดิมใช้งานได้อยู่ เอาความเสี่ยงต่ำสุด

## สิ่งที่แก้ (manifest.json — client เท่านั้น)
1. `background_color`: `#F2EDE6` → `#E8691A` (บรรทัด 7) — จบ แก้บรรทัดเดียว
   iOS สร้าง launch screen พื้นส้ม + ไอคอน (โลโก้บนพื้นขาว) กลางจอให้เอง
   และโชว์ค้างจนหน้าเว็บวาดเสร็จแล้วเด้งเข้าแอพ ตาม requirement พอดี

## ขั้นตอนหลัง deploy (ต้องแจ้งทุกคนในทีม)
1. Deploy บน Vercel ตามปกติ (ไฟล์ static ไม่แตะ server)
2. บน iPhone แต่ละเครื่อง: กดค้างไอคอนเดิม → ลบบุ๊กมาร์ก
3. เปิด Safari เข้าเว็บ → Share → Add to Home Screen ใหม่
4. เปิดจากไอคอนใหม่ → เห็นพื้นส้ม + โลโก้กลางจอระหว่างโหลด
- เครื่องที่ยังไม่ re-add: ใช้แอพได้ปกติทุกอย่าง แค่ splash ยังเป็นแบบเดิม

## แผนสำรอง (ถ้าทดสอบบนเครื่องจริงแล้ว splash ไม่เป็นสีส้ม)
- iOS บางเวอร์ชันเก่าไม่อ่าน background_color จาก manifest
  → เพิ่ม `<link rel="apple-touch-startup-image">` ภาพส้ม+โลโก้ตามขนาดจอ
  (งานเพิ่มพอสมควร — ทำเฉพาะถ้าจำเป็น)
