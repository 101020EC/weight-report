# แผนงาน V6: ข้อความคำเตือน + auto-scroll คำเตือนขึ้นบนสุด

สถานะ: ✅ ทำเสร็จ + verify ในเบราว์เซอร์แล้ว

## ขอบเขตงาน (จากผู้ใช้)
1. เปลี่ยนข้อความ "พบข้อมูลไม่ครบในบางใบขน" → "ข้อมูลไม่ครบในบางใบขนสินค้า"
2. หลังกด "อ่านข้อความด้วย Gemini" ให้ auto-scroll ให้กล่องคำเตือนอยู่บนสุด (ใต้ header) ตามรูปที่ผู้ใช้แนบ

## สิ่งที่แก้ (index.html)
1. บรรทัดหัวข้อคำเตือน: `⚠️ พบข้อมูลไม่ครบในบางใบขน` → `⚠️ ข้อมูลไม่ครบในบางใบขนสินค้า`
2. `runReadTeam` — เปลี่ยนการ scroll:
   - เดิม: `scrollBelowHeader(teamResultCard)` เรียกทันที (synchronous)
   - ใหม่: หน่วง `setTimeout(...,60)` ให้ layout settle ก่อน แล้ว scroll ไปที่
     **กล่องคำเตือน** โดยตรงเมื่อแสดง (ถ้าไม่มีคำเตือน → การ์ด) → คำเตือนอยู่บนสุดเป๊ะ

## สาเหตุที่ scroll เดิมไม่ทำงาน (พบระหว่าง debug)
- ทดสอบพบว่า `scrollBelowHeader` เดิม → warning ลงเอยต่ำกว่า header 326px (ไม่เลื่อน)
- `window.scrollY` = 0 หลังเรียก → **`behavior:'smooth'` เป็น no-op ในเบราว์เซอร์ทดสอบอัตโนมัติ**
  (บนมือถือจริง smooth ทำงานปกติ) — ดังนั้นทดสอบ target math ด้วย instant scroll แทน

## การตรวจสอบ target math (instant scroll ในเบราว์เซอร์ทดสอบ)
- header height 59px → กล่องคำเตือนลงเอยที่ 67px = **ห่างใต้ header 8px** (ตรงกับ offset -8 ใน scrollBelowHeader) ✓
- ข้อความคำเตือนอัปเดตถูก: "⚠️ ข้อมูลไม่ครบในบางใบขนสินค้า" ✓
- screenshot ตรงกับรูปที่ผู้ใช้ต้องการ (คำเตือนบนสุด ใต้ header) ✓
- ไม่มี console error ✓
